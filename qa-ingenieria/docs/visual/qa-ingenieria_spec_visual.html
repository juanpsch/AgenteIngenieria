Agente QA-Ingeniería

Revisión / QA de documentación de ingeniería · email-driven · LangGraph + Neon · OpenAI SDK · Resend · Drive/Docs/Sheets

Capas del sistema

Entrada

  • Email a qa@tenant con adjunto
  • Resend Inbound + Svix base
  • Filtro por to base
  • Entrada condicional: parser vs clasificador

Workflow

  • LangGraph + checkpointer Neon base
  • Status: RECIBIDO→EN_TRIAGE→(INCOMPLETA | NO_ADMISIBLE | EN_REVISION)→ESPERA_SENIOR→OBSERVADO/APROBADO/RECHAZADO nuevo
  • Versionado por ronda nuevo

Agentes (OpenAI SDK)

  • Parser: email→tipo/disciplina/proyecto nuevo
  • Triage: admisibilidad + completitud (¿están todos los docs requeridos de la entrega?) nuevo
  • Extractor: doc→secciones + confidence nuevo
  • Revisor: checklist + hallazgos libres nuevo
  • Clasificador de respuestas nuevo

Almacenamiento

  • Informe Google Doc → PDF base
  • Sheet maestro del proyecto nuevo
  • Carpeta por caso en Drive base

Datos / config

  • Checklists/normas por disciplina en knowledge/ nuevo
  • DRIVE_ROOT + SHEET_MAESTRO_ID por tenant nuevo
  • Senior aprobador + umbrales cron base

Trazas de escenarios (table-read)

1 · Entregable correcto, observaciones menores

RECIBIDO→ parser→ validación ✓→ EN_TRIAGE→ triage: entrega completa + formato ✓→ extractor→ revisor: 2 no-bloqueantes→ ESPERA SENIOR→ aprueba + vuelca a Sheet→ APROBADO_CON_NOTAS

2 · Archivo equivocado — lo frena el gate

RECIBIDO→ parser: dice "memoria estructural"→ validación ✓→ EN_TRIAGE→ triage: el adjunto es un plano, no coincide→ devuelve al emisor con motivo→ NO_ADMISIBLE
No corre extractor/revisor. Silencioso con el emisor las primeras 2 veces; al 3er rebote del caso → notifica al dueño. Si el emisor no acuerda con el triage → escala al senior.

2b · Entrega incompleta (falta un documento requerido)

RECIBIDO→ parser: entrega "fabricación", adjunta solo el Plano→ EN_TRIAGE→ requiere Plano + Lista de materiales · falta la lista→ reclama el faltante al emisor→ INCOMPLETA→ emisor reenvía la lista [REF]→ re-triage: completa ✓→ EN_REVISION

3 · Faltan datos mínimos

RECIBIDO→ parser: sin tipo/disciplina→ validación: faltan obligatorios→ pide al emisor (no al senior)→ FALTAN_DATOS→ responde con datos → reentra

4 · Hallazgo bloqueante + ciclo de corrección

parser/validación/triage ✓→ EN_REVISION→ revisor: 1 bloqueante + baja confianza ⚑→ ESPERA SENIOR→ aprueba (dictamen OBSERVADO)→ OBSERVADO→ emisor reenvía corregido [REF]→ clasificador: corrección→ EN_TRIAGE (ronda 2)→ re-revisa → resuelto→ APROBADO_CON_NOTAS

Cron (umbrales por etapa)

Aprobación senior pendiente

  • Estado ESPERANDO_APROBACION_SENIOR
  • Recuerda a las X horas
  • Re-escala a las Y horas

Corrección del emisor pendiente

  • Estado OBSERVADO
  • Recuerda a los X días
  • Avisa al dueño a los Y días
