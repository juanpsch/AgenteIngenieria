Fase 0 · Sandbox local local/fake

Validar el agente: recibe una entrega y dictamina completa / faltan docs / no admisible.
  • Entorno uv + Python 3.12 + deps + lockfile pyproject.toml
  • Proveedor LLM configurable (OpenAI/Claude) ai_agents/provider.py
  • Estado del caso + enum Status graph/state.py
  • Checkpointer en DB local (SQLite) tools/db.py
  • Plantillas de entrega (docs requeridos + formato) knowledge/entrega_*.md
  • Fixture del catálogo por proyecto knowledge/catalogo.json
  • Tools fake: email/sheets/drive en memoria tools/*.py
  • Lectura híbrida: PDF/Office/CAD/imagen tools/docs.py
  • build_trigger_state desde archivos tools/email.py
  • Parser de la entrega ai_agents/parser.py
  • Triage: completitud + tipo/relevancia + formato ai_agents/triage.py
  • Nodos + routers (validación, triage) graph/nodes.py · edges.py
  • Ensamblado del grafo + entrada condicional graph/graph.py
  • Contador rebotes_admisibilidad (3º → avisa)
  • Smoke runner local (CLI) scripts/run_local.py
  • Sandbox: manifest + fakes + upload de archivos sandbox/*.py
  • 4 escenarios validados con LLM real (CLI) scripts/run_local.py
Lecciones: parseo LLM regex+fallback · IO en tools/ (real+fake) · un solo build_trigger_state · ai_agents/ no agents/.

Fase 1 · Revisión local/fake

Sobre una entrega completa, producir hallazgos con severidad y dictamen propuesto.
  • Extractor: secciones/datos + confidence por ítem ai_agents/extractor.py
  • Checklists por disciplina knowledge/checklist_*.md
  • Revisor: checklist + hallazgos libres → Markdown ai_agents/revisor.py
  • Estado: hallazgos[], dictamen, EN_REVISION graph/state.py
  • Nodos extractor + revisor graph/nodes.py
  • Router por severidad (bloqueante / no bloqueante) graph/edges.py
  • Marcar baja confianza para resaltar al senior
  • Validar en sandbox: caso con y sin bloqueante
Lecciones: LLM → Markdown, el código formatea (sin loops de tool-calling) · confidence por ítem → lo dudoso a humano.
